fn add_mark(&self, value: f64, position: PositionType, markup: Option<&str>)[src]

Adds a mark at value.

A mark is indicated visually by drawing a tick mark next to the scale, and GTK makes it easy for the user to position the scale exactly at the marks value.

If markup is not None, text is shown next to the tick mark.

To remove marks from a scale, use [methodGtk.Scale.clear_marks].

value

the value at which the mark is placed, must be between the lower and upper limits of the scales’ adjustment

position

where to draw the mark. For a horizontal scale, crate::PositionType::Top and crate::PositionType::Left are drawn above the scale, anything else below. For a vertical scale, crate::PositionType::Left and crate::PositionType::Top are drawn to the left of the scale, anything else to the right.

markup

Text to be shown at the mark, using Pango markup, or None

fn set_digits(&self, digits: i32)[src]

Sets the number of decimal places that are displayed in the value.

Also causes the value of the adjustment to be rounded to this number of digits, so the retrieved value matches the displayed one, if [property[crate::Scale:draw-value] (XXX: @-reference does not belong to ScaleExt!)] is true when the value changes. If you want to enforce rounding the value when [property[crate::Scale:draw-value] (XXX: @-reference does not belong to ScaleExt!)] is false, you can set [property[crate::Range:round-digits] (XXX: @-reference does not belong to ScaleExt!)] instead.

Note that rounding to a small number of digits can interfere with the smooth autoscrolling that is built into GtkScale. As an alternative, you can use [methodGtk.Scale.set_format_value_func] to format the displayed value yourself.

digits

the number of decimal places to display, e.g. use 1 to display 1.0, 2 to display 1.00, etc

fn set_format_value_func<P: Fn(&Scale, f64) -> String + 'static>(&self, func: P)[src]

func allows you to change how the scale value is displayed.

The given function will return an allocated string representing value. That string will then be used to display the scale’s value.

If [crate::None] is passed as func, the value will be displayed on its own, rounded according to the value of the [property[crate::Scale:digits] (XXX: @-reference does not belong to ScaleExt!)] property.

func

function that formats the value

user_data

user data to pass to func

destroy_notify

destroy function for user_data

fn set_has_origin(&self, has_origin: bool)[src]

Sets whether the scale has an origin.

If [property[crate::Scale:has-origin] (XXX: @-reference does not belong to ScaleExt!)] is set to true (the default), the scale will highlight the part of the trough between the origin (bottom or left side) and the current value.

has_origin

true if the scale has an origin
